# Contributing

Thanks for your interest in improving the Raffle Spinner! This project is
intentionally small and frontend-only — contributions should keep that
spirit.

## Quick start

```bash
git clone https://github.com/ievsantillan/edm-fabric-usergroup_raffle-spinner.git
cd edm-fabric-usergroup_raffle-spinner
nvm use   # picks up Node version from .nvmrc (currently 22)
npm ci
npm run dev
```

The app is served at <http://localhost:5173/>.

## Commands

| Command         | Purpose                                                         |
|-----------------|-----------------------------------------------------------------|
| `npm run dev`   | Start the Vite dev server with HMR                              |
| `npm run lint`  | ESLint over the whole repo                                      |
| `npm run test`  | Run Vitest suite (single run)                                   |
| `npm run build` | Production build into `dist/`                                   |
| `npm run preview` | Serve the production build locally on port 4173               |

## Branch & PR workflow

`main` is protected: direct pushes are blocked, and PRs need the `verify`
and `Analyze (javascript-typescript)` checks to pass before they can merge.

1. Branch off `main`: `git checkout -b feat/<short-name>` (or `fix/…`, `chore/…`, `docs/…`)
2. Commit logically — co-authored trailers are welcome.
3. Open a PR; CI runs lint + build + dependency review on every push, plus CodeQL on push/PR.
4. Squash-merge once green.

## Code conventions

- **React 19**, function components + hooks. No class components.
- **No backend.** Everything must run in the browser (no Node-only APIs at runtime).
- **State that survives refresh** goes through `src/utils/storage.js` (which version-namespaces keys).
- **Accessibility**: keep the keyboard shortcut list working, respect `prefers-reduced-motion`,
  and use `aria-live` for any dynamic announcement that affects the draw.
- **Test the pure logic.** Hook-level tests live alongside the hook (`*.test.js`).
  We don't aim for full coverage — focus on raffle correctness (winner selection,
  undo/reroll, prize assignment).

## Updating screenshots

The README screenshots are generated by Playwright. See
`scripts/capture-screenshots.mjs` for the script and prerequisites. Run it
whenever the UI changes meaningfully.
